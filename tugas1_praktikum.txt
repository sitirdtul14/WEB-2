1. jelaskan apa yang dimaksud dengan dokumen root ?
Dokumen root adalah dokumen HTML pertama yang dibuka ketika pengguna mengakses situs web. Ini adalah dokumen utama yang mengatur struktur dan konten dari situs web tersebut.
2. pada apache web server, apakah dokumen root harus terletak dalam direktori C:/xampp\htldocs? jelaskan jawaban anda
Ya, pada Apache Web Server, dokumen root biasanya terletak dalam direktori C:/xampp/htdocs. Ini adalah lokasi default yang digunakan oleh Apache untuk mencari dan menampilkan file HTML ketika pengguna mengakses situs web. Namun, dapat mengubah lokasi dokumen root sesuai dengan kebutuhan.
3. jelaskan apa yang dimaksud dengan perintah git : init,clone,remote,add,commit,push dan checkout
Git adalah sebuah sistem kontrol versi yang digunakan untuk mengelola perubahan pada kode sumber dan proyek yang besar.
- init: Perintah ini digunakan untuk menginisialisasi repositori Git baru pada direktori saat ini.
- clone: Perintah ini digunakan untuk menyalin repositori Git yang ada ke direktori baru.
- remote: Perintah ini digunakan untuk mengelola repositori jarak jauh, seperti menambahkan repositori baru, menghapus repositori, dan mengubah informasi tentang repositori.
- add: Perintah ini digunakan untuk menambahkan file atau direktori ke staging area, yang nantinya akan di-commit.
- commit: Perintah ini digunakan untuk menyimpan perubahan yang telah ditambahkan ke staging area ke repositori.
- push: Perintah ini digunakan untuk mengirim perubahan yang telah di-commit ke repositori jarak jauh.
- checkout: Perintah ini digunakan untuk mengubah versi yang sedang aktif dari repositori, seperti mengembalikan ke versi sebelumnya atau mengambil perubahan dari repositori jarak jauh.
4. jelaskan perbedaan antara GitHub dan GitLab
GitHub dan GitLab adalah dua platform yang serupa yang digunakan untuk mengelola repositori Git dan proyek. Namun, mereka memiliki beberapa perbedaan penting:
- GitHub adalah platform yang lebih besar dan terkenal, yang digunakan oleh ribuan ribu pengembang dan tim di seluruh dunia. 
- GitLab, sebaliknya, adalah platform yang lebih kecil dan lebih fokus pada pengembangan dalam perusahaan dan organisasi.
5. apa bedanya variable user ddengan variable system ?
- Variable user adalah variabel yang digunakan untuk menyimpan informasi tentang pengguna yang sedang menggunakan sistem atau aplikasi. Ini dapat mencakup informasi seperti nama pengguna, email, alamat IP, dan lain-lain. 
- Variable system, sebaliknya, adalah variabel yang digunakan untuk menyimpan informasi tentang sistem operasi atau perangkat lunak yang sedang digunakan. Ini dapat mencakup informasi seperti versi sistem operasi, jenis prosesor, dan jumlah memori yang tersedia.
6. apa persamaan variable system dengan variable constant ?
Variable system dan variable constant adalah kedua variabel yang digunakan untuk menyimpan informasi, tetapi mereka memiliki perbedaan penting dalam penggunaan dan fungsinya.
Variable system adalah variabel yang dapat diubah atau diubah setiap saat selama program berjalan. Ini digunakan untuk menyimpan informasi yang mungkin berubah selama program berjalan, seperti nilai yang diinputkan oleh pengguna atau hasil dari operasi yang dilakukan oleh program.
